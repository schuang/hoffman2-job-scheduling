Hoffman2 Cluster Components¶

Use scenarios¶

  • The $HOME directory is for storing your source code, scripts, documents and small data files

    • Data is automatically backed up by the system

    • Be mindful for the 40GB size limit

  • The $SCRATCH directory is useful for running jobs

    • You are responsible to copy the useful data away before they are purged in 2 weeks.

  • The $TMPDIR directory, local to a compute node and not visible from other compute nodes and login nodes, is useful for use cases that can take advantage of very fast disk I/O. This directory only exists during the runtime of a job.

Examples of the directory names:

  • $HOME: /u/home/b/bruin

  • $SCRATCH: /u/scratch/b/bruin

  • $TMPDIR: /work/1234.1.pod_smp.q (different for different jobs, on different compute nodes)

  • Purchased storage: /u/project/PI_name/...

Note

It is advisable to use the environment variable names, such as $SCRATCH, in your job scripts instead of “hard-wiring” the full paths.

CPU architectures¶

Hoffman2 Cluster consists of hundreds of compute nodes. The compute nodes were purchased at different times. Consequently, there is a variety of CPU models associated with the compute nodes.

To see a list of compute nodes, use the command:

qhost

Quiz¶

  1. Which file system has has the fastest I/O speed?

    • $HOME directory,

    • $SCRATCH directory,

    • $TMP directory.

Answer

$TMP

  1. Where can you run your computational programs?

    • login node,

    • compute node,

    • In an qrsh interactive session,

Answer

Compute node or interactive session. NOT on the login node!

  1. Which file systems are visible across all login and compute nodes?

    • $HOME directory

    • $SCRATCH directory

    • $WORK directory

Answer

$Home and $SCRATCH. Work is only locally visible on a compute node.
