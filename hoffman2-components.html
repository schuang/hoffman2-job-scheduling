Hoffman2 Cluster Components¶

  • Knowing what computing resources are available are essential to successfully request adequate computing resource.

  • Such understanding helps reducing unnecessary wait time, avoid common mistakes, or know where to look when things do not work as expected.

Login nodes¶

When you use “SSH” to log in Hoffman2 Cluster, you are on one of the login nodes. As instructed in the Hoffman2 Cluster documentation, you are not supposed to run computations on the login nodes. The login nodes are primarily used for editing files, submit jobs or inspect job status.

File systems¶

Use scenarios¶

  • The $HOME directory is for storing your source code, scripts, documents and small data files

    • Data is automatically backed up by the system

    • Be mindful for the 40GB size limit

  • The $SCRATCH directory is useful for running jobs

    • You are responsible to copy the useful data away before they are purged in 2 weeks.

  • The $TMPDIR directory, local to a compute node and not visible from other compute nodes and login nodes, is useful for use cases that can take advantage of very fast disk I/O. This directory only exists during the runtime of a job.

Exercises¶

Try the following commands from your Hoffman2 login terminal.

  1. Show your home directory path:

    echo $HOME

  2. Show your scratch path:

    echo $SCRATCH

Examples¶

  • $HOME: /u/home/b/bruin

  • $SCRATCH: /u/scratch/b/bruin

  • $TMPDIR: /work/1234.1.pod_smp.q (different for different jobs, on different compute nodes)

  • Purchased storage: /u/project/PI_name/...

CPU types¶

Hoffman2 Cluster consists of hundreds of compute nodes. The compute nodes were purchased at different times. Consequently, there is a variety of CPU models associated with the compute nodes.

To see a list of compute nodes, use the command:

qhost

As of 2022, most of the compute nodes have Intel gold- and Intel E5-class CPUs. Depending on your account type, you may have access to a subset of the listed compute nodes.

To see only the Intel gold nodes, use the command:

qhost -l arch=intel-gold\*

The \ is significant, used as the escape character.

Similarly, the following command lists the E5-class nodes:

qhost -l arch=intel-E5\*

Column definitions¶

$ qhost -h n1866
HOSTNAME     ARCH         NCPU NSOC NCOR NTHR NLOAD  MEMTOT  MEMUSE  SWAPTO  SWAPUS
-----------------------------------------------------------------------------------
...
n1866        intel-gold-62  36    2   36   36  0.91  188.6G   11.3G    1.9G   64.7M
...

Some of the key columns:

  • HOSTNAME: the compute node’s name

  • ARCH: the CPU type (note: it is truncated, unfortunately.)

  • NCOR: the number of cores on the compute node

  • MEMTOT: total memory of the node

Later on in the tutorial, we will learn how to specify CPU/memory resources, if needed, in a job submission.

GPU nodes¶

There are a number of GPU nodes to support GPU computing.

See the GPU module for more details.

InfiniBand Networking¶

Most of Hoffman2 Cluster’s compute nodes are connected by Infiniband (IB) networking, which is suitable for running MPI-style parallel jobs across compute nodes efficiently.

It is important to use an MPI library that supports Infiniband to take advantage of its high-speed and low latency features. If the MPI code uns across Ethernet networking, it may still work but the general communication speed might slower by an order of magnitude.

See the MPI module for more information.

Quiz¶

  1. Which file system has has the fastest I/O speed?

    • $HOME directory,

    • $SCRATCH directory,

    • $TMP directory.

Answer

$TMP

  1. Where can you run your computational programs?

    • login node,

    • compute node,

    • In an qrsh interactive session,

Answer

Compute node or interactive session. NOT on the login node!

  1. Which file systems are visible across all login and compute nodes?

    • $HOME directory

    • $SCRATCH directory

    • $WORK directory

Answer

$Home and $SCRATCH. Work is only locally visible on a compute node.
