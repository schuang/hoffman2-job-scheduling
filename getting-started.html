Getting Started¶

Before we start discussing the details of job scheduling…

Get an account¶

Every user needs an account to log into Hoffman2 cluster. If you do not have a Hoffman2 account, use this page to apply for one. The process involves finding a faculty (typically your advisor or professor), called a “sponsor” in the account application process, who will endorse/approve your account application.

Getting an Hoffman2 account is free of charge.

Once the application process is completed, you will receive two pieces of information from this step:

  • Your Hoffman2 Cluster username

  • Your Hoffman2 Cluster password

ssh login¶

You will use the secure shell (ssh) to connect to Hoffman2 Cluster. In a Linux, Mac or Windows terminal, or something similar, run

ssh your_user_name@hoffman2.idre.ucla.edu

ssh demo

and enter your Hoffman2 Cluster password when prompted. The your_user_name above must be replaced with your Hoffman2 Cluster username.

If all steps above are successful, you should be on one of the login nodes.

Note

Use Hoffman2 Cluser username and password to log in (ssh) Hoffman2 Cluster.

They are NOT the same as the Grid username/password or your local computer’s username/password.

Text editing¶

Job submission involves writing job script (a shell script saved as a text file) that specifies the job parameters and the runtime environment. Therefore, it is useful to know how to edit text files on Hoffman2 cluster.

The recommended way is to edit the text file on Hoffman2 Cluster directly, using a text editors such as nano, vim or emacs:

nano demo

  • nano is quite easy to use, highly recommended for new users.

  • vim and emacs are very powerful; mastering them helps you become more efficient in handling text files.

Alternatively, if you must, you could edit the text file on your local computer and upload it via ssh to Hoffman2 Cluster. This approach works, although it may be cumbersome.

Note

When transferring text files to Hoffman2 cluster, be aware of the differences of the end-of-line characters between Windows (\r\n) and Linux (\n). For example, a shell script file containing \r\n will not work on Hoffman2 cluster.

Job scheduler¶

  • Hoffman2 cluster’s job scheduler is Altair Grid Engine.

  • The job scheduler’s commands and syntax provide a way for the users to request computing resources.

Note

Working definitions:

  • Job: A job is the piece of computation that the user wishes to run.

  • Job parameters: Example parameters include runtime limit and the memory size. There are many others (more on this later).

Reference: Altair Grid Engine user guide

Tip

Understanding Hoffman2 cluster’s components, the scheduler’s parameters and how to set up the runtime environment are essential to successfully submit and run the user’s jobs.

In practice, such understanding may help reducing unnecessary wait time, avoid common mistakes, or know where to look when things do not work.

Quiz¶

  1. Which file system has has the fastest I/O speed?

    • $HOME directory,

    • $SCRATCH directory,

    • $TMP directory.

Answer

$TMP

  1. Where can you run your computational programs?

    • login node,

    • compute node,

    • In an qrsh interactive session,

Answer

Compute node or interactive session. NOT on the login node!

  1. Which file systems are visible across all login and compute nodes?

    • $HOME directory

    • $SCRATCH directory

    • $WORK directory

Answer

$Home and $SCRATCH. Work is only locally visible on a compute node.
